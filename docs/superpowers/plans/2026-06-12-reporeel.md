# RepoReel Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A browser-only app that turns any git repo's history into a cinematic WebGL "code galaxy" movie with scrubbing playback and client-side video export.

**Architecture:** Next.js 16 static export hosts a fully client-side app. Pure, unit-tested engines (timeline, scene, layout simulation) feed a hand-written WebGL2 renderer (instanced stars, GPU trails, bloom post-process). Git history arrives via three adapters (local isomorphic-git in a worker, GitHub REST, pre-baked demo JSON) normalized to one `CommitTimeline` type. Export uses WebCodecs+mp4-muxer with MediaRecorder fallback.

**Tech Stack:** Next.js 16 · TypeScript · Tailwind v4 · raw WebGL2 + GLSL · Web Workers · isomorphic-git · mp4-muxer · Vitest

**Spec:** `docs/superpowers/specs/2026-06-12-reporeel-design.md`

## Core types (single source of truth, defined in Task 1, used everywhere)
```ts
// lib/types.ts
export type Source = "local" | "github" | "demo";
export type ChangeType = "add" | "modify" | "delete" | "rename";

export interface FileChange { path: string; type: ChangeType; delta: number; toPath?: string }
export interface Commit { hash: string; author: string; date: number; message: string; changes: FileChange[] }
export interface CommitTimeline { repo: { name: string; source: Source }; commits: Commit[] } // commits sorted by date asc

// timeline engine output
export interface StarLife { id: number; path: string; lang: string; birth: number; death: number | null; sizeByTime: [number, number][] } // [t, cumulativeDelta]
export interface SupernovaEvent { t: number; starIds: number[]; magnitude: number; message: string; author: string }
export interface CometPath { author: string; hops: { t: number; starId: number }[] }
// dirs: unique directory list; starDirs[i] = index into dirs for star i (plain data, safe to postMessage)
export interface Timeline { stars: StarLife[]; supernovas: SupernovaEvent[]; comets: CometPath[]; t0: number; t1: number; dirs: string[]; starDirs: number[] }

// scene sampling
export interface SceneState { t: number; aliveStarIds: number[]; sizes: Float32Array; pulses: Float32Array; activeSupernovas: { starIds: number[]; age: number; magnitude: number }[]; cometPositions: { author: string; fromStar: number; toStar: number; progress: number }[] }

// layout
export interface LayoutFrame { positions: Float32Array /* x,y per star id */; version: number }
```

### Task 6: Engine effects: supernovas, comet trails, bloom

**Files:** `engine/effects.ts`, `engine/post.ts`, extend `engine/shaders.ts`, wire in `engine/renderer.ts`.

- [ ] `engine/effects.ts`:
  - Supernova rings: per active supernova, draw an expanding ring at the centroid of its `starIds` positions: triangle-fan circle outline shader, radius `age * 220 * magnitude`, alpha `(1 - age)`, color theme accent.
  - Comet trails: per comet, a strip of the last `TRAIL_LEN = 24` interpolated positions (lerp fromStar→toStar by progress, history kept on CPU ring buffer), drawn as `gl.TRIANGLE_STRIP` with width tapering and alpha fading to 0; comet head = one bright star point.
  - Star death collapse: when a star's `death` falls within the last `COLLAPSE_MS = 1200` of timeline time, draw it shrinking to 0 with an inverted dark halo ring (the spec's black-hole moment); scene sampler already excludes it after death, so effects read `death` directly from `Timeline.stars`.
  - Camera shake: `camera.shake(magnitude)` adds a decaying (0.85/frame) random offset ≤ 6px * magnitude to the view matrix; renderer calls it when a supernova with `magnitude > 0.6` activates.
- [ ] `engine/post.ts`: bloom pipeline:
  1. render scene to FBO A (HALF float color texture, fallback UNSIGNED_BYTE if `EXT_color_buffer_half_float` missing),
  2. bright-pass into half-res FBO B (`max(color - 0.6, 0) * 1.6`),
  3. separable Gaussian blur B→C→B (2 passes, 9-tap),
  4. composite to screen: `scene + bloom * u_bloomStrength(1.2)`.
  - Expose `setQuality("high" | "low")`: low skips bloom (mobile/weak GPU fallback when FPS EMA < 40 for 3s, auto-downgrade with console.info).
- [ ] Wire into `Renderer.frame()`: scene→FBO, effects→same FBO, post→screen. `/perf` page gains a supernova-every-second + 20 comets stress toggle; verify ≥ 50 FPS, note in commit body.
